# Задание: Работа с Git - Pull, Push и управление изменениями

## Цель задания
Освоить ключевые команды Git для синхронизации локального и удаленного репозиториев, научиться вносить изменения в файлы и доставлять их на удаленный репозиторий, используя как командную строку, так и интерфейс Visual Studio Code.

## Детализация задач

### 1. Изучение основных команд Git

#### Команда `git pull`
- **Назначение**: Получение и объединение изменений из удаленного репозитория в текущую ветку
- **Синтаксис**: `git pull [<опции>] [<репозиторий>] [<ветка>]`
- **Основные опции**:
  - `--rebase`: Перебазирование вместо слияния
  - `--no-commit`: Не создавать автоматический коммит при слиянии
  - `--ff-only`: Обновлять только если возможен fast-forward

#### Команда `git push`
- **Назначение**: Отправка локальных коммитов в удаленный репозиторий
- **Синтаксис**: `git push [<опции>] [<репозиторий>] [<ветка>]`
- **Основные опции**:
  - `-u` или `--set-upstream`: Установить upstream ветку
  - `--force-with-lease`: Безопасная альтернатива `--force`
  - `--tags`: Отправлять теги вместе с коммитами

#### Команда `git push --force`
- **Назначение**: Принудительная отправка изменений (перезаписывает историю)
- **Предупреждение**: Использовать с осторожностью, может удалить чужие коммиты
- **Альтернативы**:
  - `git push --force-with-lease` - более безопасный вариант
  - `git push --force-if-includes` - проверяет актуальность локальной копии

### 2. Работа с изменениями файлов

#### Базовый рабочий процесс:
1. **Внесение изменений** в файлы проекта
2. **Добавление файлов** в индекс: `git add <файлы>`
3. **Создание коммита**: `git commit -m "описание изменений"`
4. **Отправка на удаленный репозиторий**: `git push`

#### Полезные команды для управления изменениями:
- `git status` - просмотр состояния файлов
- `git diff` - просмотр изменений
- `git log` - просмотр истории коммитов
- `git restore` - отмена изменений в файлах
- `git reset` - отмена индексации файлов

### 3. Практические задачи

#### Через командную строку (терминал):
1. Клонировать существующий репозиторий
2. Создать новую ветку для работы
3. Внести изменения в несколько файлов
4. Добавить файлы в индекс и создать коммит
5. Получить актуальные изменения с удаленного репозитория (`git pull`)
6. Отправить изменения на удаленный репозиторий (`git push`)
7. Попробовать принудительную отправку (`git push --force`) на тестовой ветке

#### Через Visual Studio Code:
1. Открыть проект в VS Code
2. Использовать встроенный интерфейс Git:
   - Панель "Source Control" (Ctrl+Shift+G)
   - Индикатор состояния файлов
   - Визуальное сравнение изменений
3. Выполнить аналогичные операции через GUI:
   - Stage changes
   - Commit changes
   - Push to remote
   - Pull from remote
   - Resolve merge conflicts
